Jekyll and Yeoman

22 Jul 2014

Yet again, I’ve restarted my blog, with a super-plain design and really basic functionality1. I’m trying to adhere more to the 80 / 20 principle - not the Pareto Principle - but the idea that one should just get the project to 80%, set it free, and then improve later. Operational and improving is better than hidden and perfecting. I swear I heard it somewhere reputable.

So then, this blog is out in the open. I’m hopeful I will be encouraged to write more, even if just for good practice.

There is a glaring 20% that needs work at some stage though:

  • responsiveness and mobile devices. The site title doesn’t scale for mobiles, and the footnoting jQuery hackery needs to basically be disabled as well for these devices (Bootstrap should just wrap the footnotes under the content). Post dates on the homepage also looky scruffy.
  • colours and fonts: a background colour that isn’t white, and a bit of experimentation with relative font sizings
  • test out images in posts, see if they will behave responsibly
  • and there are still several issues with using the given gruntfile to generate the dist version of the site currently in use
    • the main README and my CNAME file stubbornly won’t copy across on a new build
    • can’t get google fonts working with the Sass - CSS - minification process, just manually linking currently
    • font awesome font files, if revved, won’t load correctly, so have just ignored revving them

But on the bright side, none of these are back-breaking issues for now. One thing that I did discover was that my JS usemin block wouldn’t concatenate my bower components files correctly, unless I linked these files first within the block. Weird, I don’t quite get it and in fact it’s likely that I’ve done something wrong somewhere. Linking the bower component first does work though, so hey.
